1. What is Network Security

  • Definition:

    Protection of computer networks and data from unauthorized access, misuse, modification, or denial of service.

  • Goal: Ensure Confidentiality, Integrity, and Availability (the CIA triad).
  • Example: Using firewalls, encryption, and secure protocols to protect data transfer.

2. Key Principles of Security (CIA + A + N)

  1. Confidentiality:

    Only authorized users can access data.

    ‚Üí Example: Data encryption.

  1. Integrity:

    Data cannot be modified or tampered with.

    ‚Üí Example: Hashing.

  1. Availability:

    Data and services are accessible when needed.

    ‚Üí Example: Redundant servers.

  1. Authentication:

    Verifying user identity.

    ‚Üí Example: Login password or biometric.

  1. Non-repudiation:

    Sender cannot deny sending a message.

    ‚Üí Example: Digital signatures.

3. Threats and Vulnerabilities

  • Threat: Any potential danger that can exploit a weakness.

    ‚Üí Example: Hacker, malware, phishing.

  • Vulnerability: A weakness in system design or implementation.

    ‚Üí Example: Weak password, unpatched software.

  • Risk: Probability of threat exploiting a vulnerability.

7. Symmetric and Asymmetric Cryptography

a) Symmetric Key Cryptography

  • Same key used for encryption and decryption.
  • Fast but less secure (key sharing problem).
  • Examples: AES, DES, Blowfish.
  • Used in: Data encryption, secure channels.

b) Asymmetric Key Cryptography

  • Uses public key for encryption and private key for decryption.
  • Slower but more secure (no key sharing).
  • Examples: RSA, ECC, Diffie‚ÄìHellman.
  • Used in: Digital signatures, SSL/TLS, HTTPS.

9. IPsec (Internet Protocol Security)

  • Definition: Framework to secure IP communication by authenticating and encrypting packets at the Network Layer.
  • Components:
    1. AH (Authentication Header): Ensures integrity and authentication.
    1. ESP (Encapsulating Security Payload): Ensures confidentiality and integrity.
  • Used in: VPNs (Virtual Private Networks).

10. HTTPS (HyperText Transfer Protocol Secure)

  • Secure version of HTTP that uses SSL/TLS encryption.
  • Ensures:
    • Confidentiality (data encrypted)
    • Integrity (data not changed)
    • Authentication (valid server)
  • Port: 443
  • Example: Used in all secure websites (padlock icon üîí).

11. SSL / TLS (Secure Sockets Layer / Transport Layer Security)

  • Protocol for securing data in transit between client and server.
  • Works at Transport Layer (above TCP).
  • Uses public key cryptography for key exchange and symmetric keys for data encryption.
  • TLS is the improved, modern version of SSL.
  • Steps:
    1. Handshake (key exchange)
    1. Session encryption
    1. Secure communication

12. IDS (Intrusion Detection System)

  • Definition: Monitors network traffic for suspicious activity or known attacks.
  • Types:
    1. Network-based IDS (NIDS): Monitors entire network.
    1. Host-based IDS (HIDS): Monitors one machine.
  • Functions: Detect, alert, and log ‚Äî does not block.
  • Example: Snort.

13. Firewall

  • Definition: Network security device/software that filters incoming and outgoing traffic based on predefined rules.
  • Types:
    1. Packet Filtering Firewall: Checks headers only.
    1. Stateful Firewall: Tracks active connections.
    1. Proxy Firewall: Acts as an intermediary.
    1. Next-Gen Firewall (NGFW): Uses deep packet inspection + IDS/IPS features.
  • Purpose: Prevent unauthorized access to/from network.
