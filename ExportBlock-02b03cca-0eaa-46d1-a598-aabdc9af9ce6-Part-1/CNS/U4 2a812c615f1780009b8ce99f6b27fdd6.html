1. Process-to-Process Delivery

Definition:

While the network layer delivers packets host-to-host, the transport layer ensures data delivery process-to-process (i.e., between applications running on different systems).

Example:

  • Web browser (client process) ‚Üî Web server (server process).
  • Email app ‚Üî Mail server.

Key Protocols:

  • TCP (Transmission Control Protocol): Reliable, connection-oriented.
  • UDP (User Datagram Protocol): Fast, connectionless.

2. Services Provided by Transport Layer

  1. Process-to-Process Delivery:
    • Data sent directly between application processes using port numbers.
  1. Connection Control:
    • Establish, maintain, and terminate connections (TCP only).
  1. Error Control:
    • Detect and retransmit lost or corrupted data.
  1. Flow Control:
    • Prevent sender from overwhelming the receiver.
  1. Segmentation and Reassembly:
    • Divide large messages into smaller segments, then reassemble at destination.
  1. Multiplexing and Demultiplexing:
    • Multiple apps use one network connection simultaneously (e.g., web + mail).
  1. Reliable Delivery:
    • TCP ensures all data reaches destination in correct order.

3. Socket and Addressing

Socket:

A combination of IP address and Port number that uniquely identifies a connection between two processes.

Socket Format:

<IP address, Port number>

Example:

  • Client: <192.168.1.2, 5050>
  • Server: <192.168.1.10, 80>

6. Flow Control

Definition:

A mechanism to ensure that the sender does not send data faster than the receiver can process.

Technique:

  • Sliding Window Protocol:
    • Data is sent in ‚Äúwindows‚Äù (chunks).
    • Receiver sends acknowledgment (ACK) after receiving a set of packets.
    • Window size adjusts dynamically.

Analogy:

Like talking ‚Äî you pause to ensure the listener understands before continuing.

7. Buffering

Definition:

Temporary storage area used to smooth the speed difference between sender and receiver.

Types:

  1. Single Buffering: One buffer used for sending or receiving data.
  1. Double Buffering: Two buffers alternate ‚Äî one for sending, one for filling.
  1. Circular Buffering: Multiple buffers used cyclically for continuous data flow.

Purpose:

  • Handles bursty traffic.
  • Prevents data loss due to speed mismatch.
  • Ensures smooth, efficient transmission.
