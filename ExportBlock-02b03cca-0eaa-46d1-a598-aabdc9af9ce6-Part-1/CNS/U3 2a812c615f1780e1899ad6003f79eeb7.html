1. Network Layer ‚Äî Overview

Definition:

The Network Layer is responsible for routing and delivering packets from the source to destination across multiple networks.

Main Functions:

  1. Logical addressing (IP addresses)
  1. Routing ‚Äî finding best path
  1. Packet forwarding
  1. Fragmentation & reassembly
  1. Error handling and congestion control

‚öôÔ∏è 2. Switching Techniques

Switching decides how data travels through the network from source to destination.

a) Circuit Switching

  • A dedicated communication path is established between sender and receiver before data transfer.
  • All data follows the same path.
  • Example: Telephone network.

Phases:

  1. Connection setup
  1. Data transfer
  1. Connection termination

Advantages:

  • Guaranteed delivery, no congestion.
  • Constant transmission rate.

Disadvantages:

  • Wastes bandwidth if no data is sent.
  • Setup delay.

b) Message Switching

  • Entire message is treated as one unit.
  • Each intermediate node stores the whole message, then forwards it ‚Äî called Store and Forward.
  • No dedicated path.

Advantages:

  • Efficient channel utilization.
  • Can handle variable message sizes.

Disadvantages:

  • High delay (must store full message at each hop).
  • Requires large memory at each node.

c) Packet Switching

  • Message divided into small packets.
  • Each packet sent independently through the network.
  • Packets may take different paths.

Types:

  1. Datagram packet switching: Each packet routed independently (e.g., Internet).
  1. Virtual circuit switching: Path established logically before sending (e.g., Frame Relay, ATM).

Advantages:

  • Fast, efficient, low delay.
  • Bandwidth sharing.

Disadvantages:

  • Packet loss, disorder possible.
  • Requires reassembly at destination.

üß≠ 3. Routing Algorithms

Routing determines the best path for packets from source to destination.

a) Static Routing

  • Manual route setup by network admin.
  • Path remains fixed unless changed manually.

Advantages: Simple, secure.

Disadvantages: Not adaptive to network changes or failures.

Used in: Small networks.

b) Dynamic Routing

  • Routers automatically update routes based on network conditions.
  • Uses routing algorithms and protocols.

Advantages: Adapts to changes.

Disadvantages: Requires more CPU, memory.

Used in: Large, complex networks.

c) Distance Vector Routing

  • Each router maintains a table (vector) of minimum distances (hops) to every other router.
  • Routers share info periodically with neighbors.

Algorithm Used: Bellman-Ford.

Example Protocol: RIP (Routing Information Protocol).

Advantages: Simple, easy to implement.

Disadvantages: Slow convergence, routing loops possible.

üåê 4. Routing Protocols

Routing protocols define how routers exchange routing information to determine optimal paths.

a) RIP (Routing Information Protocol)

  • Type: Distance vector protocol.
  • Algorithm: Bellman-Ford.
  • Metric: Hop count (max = 15).
  • Update Interval: Every 30 seconds.

Advantages: Simple, easy setup.

Disadvantages: Slow convergence, not suitable for large networks.

b) OSPF (Open Shortest Path First)

  • Type: Link-state protocol.
  • Algorithm: Dijkstra‚Äôs Shortest Path.
  • Metric: Cost (based on bandwidth).
  • Updates: Only when topology changes.

Advantages: Fast, supports large networks.

Disadvantages: More complex and memory-intensive.

c) BGP (Border Gateway Protocol)

  • Type: Path vector protocol.
  • Used between autonomous systems (AS) ‚Äî i.e., between different organizations.
  • Used in: Internet backbone routing.

Advantages: Handles large, complex routing tables.

Disadvantages: Slow convergence, complex configuration.

d) MPLS (Multiprotocol Label Switching)

  • Uses labels instead of IP addresses for routing.
  • Each packet assigned a label; routers forward packets based on labels (not full IP lookup).

Advantages:

  • Faster packet forwarding.
  • Supports QoS and traffic engineering.
  • Can work with different protocols (IP, ATM, Frame Relay).
