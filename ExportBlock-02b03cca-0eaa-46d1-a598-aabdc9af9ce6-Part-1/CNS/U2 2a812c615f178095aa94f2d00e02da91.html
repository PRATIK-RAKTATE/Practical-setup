1. Design Issues of the Network Layer

Definition:

The network layer is responsible for transferring packets from source to destination across multiple networks.

Its design issues define how efficiently and accurately data travels through the network.

Main Design Issues:

  1. Addressing:
    • Assigning unique network addresses (like IP addresses) to each device.
  1. Routing:
    • Finding the best path for data packets.
    • Algorithms: Distance Vector, Link State.
  1. Packetizing:
    • Dividing data into small packets for transmission.
    • Each packet includes source and destination info.
  1. Fragmentation & Reassembly:
    • Splitting large packets to fit smaller network frames.
    • Reassembled at destination.
  1. Error Handling:
    • Detect and manage corrupted or lost packets.
  1. Congestion Control:
    • Avoid overload by regulating data flow.
  1. Quality of Service (QoS):
    • Ensure reliable delivery for time-sensitive data (e.g., video calls).
  1. Security:
    • Prevent unauthorized access and data tampering.

2. Services Provided to the Network Layer

Network layer provides services to the Transport Layer above it.

Main Services:

  1. Connectionless Service:
    • Each packet treated independently (no setup).
    • Example: IP (Internet Protocol).
    • Faster, but no guaranteed delivery.
  1. Connection-Oriented Service:
    • Establishes a virtual path before sending data.
    • Example: ATM, Frame Relay.
    • Reliable delivery and sequencing.
  1. Routing Services:
    • Determines optimal route to reach destination.
  1. Error Reporting & Control:
    • Uses ICMP messages for error notification.
  1. Packet Forwarding:
    • Transfers packets between routers until destination is reached.

3. Error Detection and Error Correction

Definition:

During transmission, noise or interference can alter data bits.

Error detection and correction ensure reliable communication.

Types:

  1. Error Detection:
    • Identifies if an error occurred during transmission.
    • Techniques:
      • Parity Bit
      • Checksum
      • CRC (Cyclic Redundancy Check)
  1. Error Correction:
    • Detects and corrects the error at receiver side.
    • Techniques:
      • Hamming Code
      • Forward Error Correction (FEC)

4. Parity Bit

Definition:

Simplest method for error detection by adding an extra bit (parity bit) to make total 1s even or odd.

Types:

  • Even Parity: Total 1s (including parity bit) = even.
  • Odd Parity: Total 1s (including parity bit) = odd.

5. Hamming Code

Definition:

An error correction code that can detect up to 2-bit errors and correct 1-bit error.

Working Principle:

  1. Adds redundant bits (parity bits) at positions that are powers of 2.

  1. Each parity bit covers specific positions of the data bits.
  1. On receiving, parity bits are rechecked to find the error position.
