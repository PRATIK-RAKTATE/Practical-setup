1. LAN (Local Area Network)

  • Connects computers in a small area (office, school, lab).
  • Range: Up to a few kilometers.
  • Ownership: Private.
  • Devices Used: Switch, Hub, Router.
  • Advantages: High speed, low cost, easy maintenance.
  • Example: College computer lab.

2. MAN (Metropolitan Area Network)

  • Connects multiple LANs across a city or large campus.
  • Range: 50‚Äì60 km.
  • Ownership: Public or private.
  • Advantages: Medium cost, connects organizations in one city.
  • Example: Cable TV network, city Wi-Fi.

3. WAN (Wide Area Network)

  • Connects multiple LANs or MANs globally.
  • Range: Unlimited (worldwide).
  • Ownership: Public networks, telecom lines.
  • Devices Used: Routers, Satellites, Fiber optics.
  • Example: The Internet.

4. Wireless Networks

  • Uses radio, microwave, or infrared signals (no cables).
  • Types:
    • WPAN ‚Üí Bluetooth
    • WLAN ‚Üí Wi-Fi
    • WWAN ‚Üí Mobile Networks (4G/5G)
  • Advantages: Flexible, mobile, easy setup.
  • Disadvantages: Signal interference, less secure.

5. Protocol Design Issues in Network Layer

  • Network Layer Function: Packet delivery, routing, addressing.
  • Design Issues:
    1. Addressing: Unique IPs for devices.
    1. Routing: Find best path to destination.
    1. Packetizing: Divide data into packets.
    1. Fragmentation/Reassembly: Handle large packets.
    1. Error Control: Manage lost/corrupted packets.
    1. Congestion Control: Avoid network overload.
    1. Security: Protect data in transmission.

6. Network Models

OSI Model (7 Layers)

  1. Physical ‚Äì Transmission of bits
  1. Data Link ‚Äì Error detection (Frames)
  1. Network ‚Äì Routing (Packets)
  1. Transport ‚Äì End-to-end delivery (Segments)
  1. Session ‚Äì Manage sessions
  1. Presentation ‚Äì Data translation, encryption
  1. Application ‚Äì User interface (HTTP, FTP)

TCP/IP Model (4 Layers)

  1. Network Access ‚Äì Physical & Data link
  1. Internet ‚Äì Routing (IP)
  1. Transport ‚Äì Communication (TCP/UDP)
  1. Application ‚Äì Services (HTTP, DNS, SMTP)

7. Network Topologies

  • Bus: Single cable shared by all devices ‚Üí simple, cheap.
  • Star: All nodes connected to a central switch ‚Üí reliable, scalable.
  • Ring: Devices connected in a loop ‚Üí data flows one direction.
  • Mesh: Every device connected to every other ‚Üí high reliability.
  • Hybrid: Combination of topologies.

8. Network Architectures

Client-Server:

  • Central server provides data/services to clients.
  • Example: Web server & browser.
  • Advantages: Centralized control, security.
  • Disadvantages: Server failure affects all clients.

Peer-to-Peer (P2P):

  • All devices are equal; share data directly.
  • Example: Torrent, Bluetooth sharing.
  • Advantages: No central server, cost-effective.
  • Disadvantages: Security and data management issues.

Hybrid:

  • Combination of Client-Server + P2P.
  • Example: Modern enterprise systems, collaboration apps.

9. Networking Devices

Bridge

  • Connects two LAN segments (same protocol).
  • Filters traffic using MAC addresses.
  • Works at: Data Link Layer (Layer 2).

Router

  • Connects multiple networks.
  • Chooses best path for data (Routing).
  • Works at: Network Layer (Layer 3).

Gateway

  • Connects different network types or architectures.
  • Performs protocol translation.
  • Works at: All layers (mainly Application Layer).
