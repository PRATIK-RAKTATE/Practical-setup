1. LAN

  • Small area: Office/school/lab
  • Range: Few km
  • Ownership: Private
  • Devices: Switch, Hub, Router
  • Pros: High speed, low cost, easy maintain
  • Ex: College lab network

2. MAN

  • Connects multiple LANs: City/large campus
  • Range: 50‚Äì60 km
  • Ownership: Public/Private
  • Pros: Medium cost, city orgs connected

3. WAN

  • Global connectivity: LANs/MANs across regions
  • Range: Unlimited (worldwide)
  • Ownership: Public/telecom
  • Devices: Router, Satellite, Fiber
  • Ex: Internet

4. Wireless Networks

  • Signals: Radio/microwave/infrared
  • Types: WPAN (Bluetooth), WLAN (Wi-Fi), WWAN (4G/5G)
  • Pros: Flexible, mobile, easy setup
  • Cons: Interference, less secure

2. Services Provided (Network Layer ‚Üí Transport)

  • Connection-Oriented: ATM/Frame Relay ‚Üí virtual path, reliable
  • Routing: Optimal path determination

3. Error Detection vs Correction

  • Detection: Identify errors (Parity, Checksum, CRC)
  • Correction: Fix errors (Hamming, FEC)

4. Parity Bit (Error Detection)

  • Extra bit: Even/odd total 1s (inc parity)
  • Example: Data `1010001` (3 1s ‚Üí odd)
    Even parity: `10100011` (4 1s)
    Odd parity: `10100010` (3 1s)
  • Limitation: Detects only single-bit errors

5. Hamming Code (Error Correction)

  • Corrects 1-bit errors, detects 2-bit errors
  • Parity bits at positions: 2‚Åø (1, 2, 4, 8‚Ä¶)
  • Example: 4 data bits + 3 parity bits ‚Üí `P1 P2 D3 P4 D2 D1 D0`
  • Error loc: Binary parity result = bit position (flip to fix)
  • Advantage: Auto-corrects single-bit errors

‚è≥ Network Layer Oral Cheat Sheet (30-sec Prep)

1. Network Layer

  • Function: Route/deliver packets across networks.
  • Core tasks: Logical addr, routing, forwarding, frag/reasm, error handling, congestion control.

2. Switching Techniques

  • Circuit Switching:
  • Fixed path (dedicated); setup/transfer/terminate phases. Ex: Telephony.
  • Message Switching:
  • Store&Forward full message. Ex: Telegraph systems.
  • Packet Switching:
  • Split data into packets (independent paths). Types: Datagram (Internet), Virtual Circuit (Frame Relay).

3. Routing Algorithms

  • Static:
  • Manual route setup (fixed). Adv: Simple. Dis: Not adaptive. Use: Small nets.
  • Dynamic:
  • Auto-update routes (adaptive). Adv: Scalable. Dis: CPU/memory heavy. Use: Large nets.

4. Routing Protocols

  • RIP (Distance Vector):
  • Bellman-Ford; metric=hop count (max 15). Adv: Simple. Dis: Slow convergence. Use: Small nets.
  • OSPF (Link State):
  • Dijkstra‚Äôs; metric=cost (bandwidth). Adv: Fast. Dis: Complex. Use: Enterprise.
  • BGP (Path Vector):
  • Inter-AS routing. Metric=AS path. Use: Internet backbone.
  • MPLS (Label Switching):
  • Labels instead of IP for routing. Adv: Faster, QoS. Use: ISP backbones.
